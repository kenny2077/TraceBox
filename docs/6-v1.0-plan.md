## Phase I: Connect the Runtime (Critical — Blocks v1.0)

**Theme:** `tracebox run -- claude` actually works end-to-end.

### I-A: Wire MCP Proxy into Orchestrator (3 days)

**Problem:** The TraceGate stdio proxy (`packages/policy/src/tracegate/proxy.py`, 353 lines) is a working subprocess relay with policy evaluation, but it runs standalone. The orchestrator doesn't launch it.

**Solution:** `tracebox run -- <agent>` launches the agent with TraceGate as an MCP stdio proxy between the agent and its tools.

```
Agent                 TraceGate Proxy             MCP Servers
  │                         │                         │
  │── JSON-RPC request ────►│── evaluate policy ─────►│
  │                         │── log to ledger ───────►│
  │◄──── response ──────────│◄──── response ──────────│
  │                         │                         │
```

**Tasks:**
- [ ] `SessionOrchestrator.run_agent_with_proxy()`: spawns agent subprocess + proxy sidecar
- [ ] Proxy forwards all `tools/call` requests to orchestrator for policy eval
- [ ] Proxy logs every request/response to ledger
- [ ] Proxy captures stdout/stderr from agent (for timeline context)
- [ ] Fallback: if agent doesn't support MCP proxy, fall back to file-watching mode
- [ ] Test: `tracebox run -- echo "hello"` creates session, logs tool call, ends session

**Files involved:** `packages/core/orchestrator.py`, `packages/policy/src/tracegate/proxy.py`

### I-B: Wire TTY Approval Prompt (2 days)

**Problem:** `packages/policy/src/tracegate/approval.py` (4.8KB) has working `/dev/tty` approval prompts, but the policy engine returns `"ask"` decision type without triggering the prompt.

**Solution:** When policy returns `decision="ask"`, the orchestrator calls `approval.prompt()` before allowing/denying.

```python
# In on_tool_call():
if decision["decision"] == "ask":
    user_choice = approval_prompt(tool_name, arguments, decision["reason"])
    if user_choice == "deny":
        decision["decision"] = "deny"
    elif user_choice == "allow_once":
        decision["decision"] = "allow"
    elif user_choice == "allow_always":
        decision["decision"] = "allow"
        approval_memory.add(tool_name, arguments_fingerprint)
```

**Tasks:**
- [ ] Import and call `tracegate.approval.prompt()` from orchestrator
- [ ] Create `ApprovalMemory` class for fuzzy-match caching
- [ ] Wire "always allow similar" into memory
- [ ] Handle Windows missing /dev/tty → auto-deny with warning
- [ ] Test: mock /dev/tty, verify approve/deny/memory works

### I-C: Wire DLP Response Redaction (1 day)

**Problem:** `packages/policy/src/tracegate/dlp.py` (3.5KB) can redact secrets from tool call arguments and responses, but the orchestrator doesn't call it.

**Solution:** Every tool call response goes through DLP before returning to agent.


**Tasks:**
- [ ] Wire `dlp_engine.redact()` into proxy response pipeline

### I-D: Wire Web Dashboard into CLI (2 days)

## Phase II: Complete the UX (High — Makes v1.0 Feel Real)

### II-A: Approval Memory with Fuzzy Matching (2 days)

**Problem:** Without memory, every "ask" requires user input. Users will disable the feature.

**Solution:** Cache approvals by (tool_name, arguments_fingerprint) so repeated similar calls auto-allow.

```python
class ApprovalMemory:
    def __init__(self, db_path: str = ".tracebox/approvals.db"):
        self.db = sqlite3.connect(db_path)
    
    def add(self, tool: str, args: dict, duration: int = 3600):
        fingerprint = self._fingerprint(args)
        expires_at = time.time() + duration
        self.db.execute("INSERT OR REPLACE INTO approvals VALUES (?, ?, ?)",
                       (tool, fingerprint, expires_at))
    
    def check(self, tool: str, args: dict) -> bool:
        fingerprint = self._fingerprint(args)
        row = self.db.execute("SELECT expires_at FROM approvals WHERE tool=? AND fingerprint=? AND expires_at>?",
                             (tool, fingerprint, time.time())).fetchone()
        return row is not None
    
    def _fingerprint(self, args: dict) -> str:
        # Normalize: strip whitespace, sort keys, hash the JSON
        return hashlib.sha256(json.dumps(args, sort_keys=True).encode()).hexdigest()[:16]
```

**Tasks:**
- [ ] Create `packages/policy/approval_memory.py`
- [ ] Wire into orchestrator's `on_tool_call()`
- [ ] Export: `tracebox policy clear-memory`
- [ ] Test: approve once, repeat call, verify auto-allowed

### II-B: Network Destination Logging (2 days)

**Problem:** TraceGate logs tool calls but doesn't specifically flag network destinations.

**Solution:** Parse tool call arguments for URLs, hostnames, IPs — log as `network_events`.

```python
def detect_network_destinations(args: dict) -> list:
    destinations = []
    for key, value in args.items():
        if isinstance(value, str):
            # Check for URLs
            urls = re.findall(r'https?://[^\s"\'<>]+', value)
            destinations.extend(urls)
            # Check for host:port patterns
            hosts = re.findall(r'([a-zA-Z0-9.-]+):(\d+)', value)
            destinations.extend([f"{h}:{p}" for h, p in hosts])
    return destinations
```

## Phase III: Polish & Ship (Medium — Quality bar for v1.0)

### III-A: Fix RippleGraph Node.js ESM Bug (1 day)

**Problem:** `packages/graph/dist/db.js` uses `require("better-sqlite3")` but `package.json` has `"type": "module"`, causing `ReferenceError: require is not defined`. Only Bun works.

**Solution:** Two options:
1. **(Recommended)** Document that RippleGraph requires Bun. Add `bun` to dependencies. Add `tracebox graph init` that auto-detects Bun and runs.
2. **(Hard)** Rewrite db.ts to use ESM `import()` for better-sqlite3.

**Tasks:**
- [ ] Add Bun detection in orchestrator
- [ ] Add friendly error: "RippleGraph requires Bun. Install: curl -fsSL https://bun.sh/install | bash"
- [ ] Add `tracebox doctor` command that checks Bun availability
- [ ] Document in README

### III-B: Real-World Rollback Testing (2 days)

**Problem:** Rollback engine passes unit tests but hasn't been tested on real agent changes.

**Solution:** Create comprehensive rollback test matrix.

| Scenario | Expected | Tested? |
|----------|----------|---------|
| Modified file | Reverse patch applied | ✅ |
| Created file | File deleted | ✅ |
| Deleted file | File restored from snapshot | ✅ |
| Renamed file | Reversed | ❌ |
| Package.json change | Warn, don't auto-revert | ❌ |
| .env change | Warn, ask confirmation | ❌ |
| File outside repo | Skip with warning | ❌ |
| Binary file change | Snapshot restore | ❌ |
| Large file (>10MB) | Skip with warning | ❌ |

**Tasks:**
- [ ] Test all 9 scenarios
- [ ] Fix any failures
- [ ] Add edge case handling (binary, large, outside-repo files)
- [ ] Update rollback report with per-file confidence scores
